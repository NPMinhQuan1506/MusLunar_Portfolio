
#  MusLunar Portfolio – Project Charter

## 1. Purpose & Vision
Create an interactive, dual-persona portfolio (Developer + Designer) that integrates high-end storytelling, e-commerce, and gamification. The platform aims to convert visitors into clients/recruiters by proving technical depth (Go/Next.js) and creative flair (3D/Animation).

## 2. Objectives
- **Duality:** Showcase both Logic (Quan) and Design (Nguyet) in a unified product.
- **Conversion:** Turn viewers → interviews (Recruiters) and freelance leads (Clients).
- **Foundation:** Build a scalable architecture for future digital product sales and blogging.

## 3. Scope
**In Scope (MVP & Full Release):**
- **Core:** Animated Intro, Role Selector, Dual-Home (Drag Interaction).
- **Showcase:** Project List/Detail with Role-based filtering.
- **Storytelling:** Interactive Timeline (Gamified milestones).
- **Commerce:** Shop module (Digital/Physical goods) with Manual/QR Payment.
- **System:** Admin Dashboard (CMS), Multi-language (VI/EN), Dark/Light/Pixel Themes.

**Out of Scope (for v1):**
- **Public Sign-up:** No community registration system (Only Admin & Recruiter Access).
- **Automated Payment:** No Stripe/PayPal integration (Manual verify only).
- **Advanced AI:** No self-learning personalization engine (Rule-based only).

## 4. Stakeholders
| Role | Name | Responsibility |
|------|------|----------------|
| **Product Owner / Lead Dev** | Quan Nguyen | Architecture, Backend (Go), Frontend Logic, Infra. |
| **Creative Director / Designer** | Nguyet Nguyen | UI/UX, Assets, Storyboarding, QA. |
| **End Users** | Clients, Recruiters | The target audience to impress. |

## 5. Constraints & Tech Stack
- **Team:** 2 members (1 Dev, 1 Des).
- **Time Commitment:** ~9h/week (Evenings & Weekends).
- **Budget:** Optimized to **~2M VND/year** (Buffer up to 5M for assets/ads).
- **Frontend:** Next.js 15, Tailwind CSS, Shadcn UI, Framer Motion, React Three Fiber.
- **Backend:** Go (Golang), Echo Framework, Sqlc.
- **Database:** PostgreSQL (Supabase).
- **Infrastructure:** Vercel (FE) + CloudFly VPS (BE).

## 7. Risks & Mitigation
| Risk | Impact | Mitigation |
|------|---------|-------------|
| **Go Learning Curve** | Medium | Stick to simple architecture (Clean Arch), use `sqlc` for safety. |
| **VPS Maintenance** | High | Automate deployment via GitHub Actions; Keep DB on Supabase (Managed). |
| **Scope Creep** | High | Strict "No New Features" policy during Sprint 1. |
| **Asset Delays** | Medium | Dev uses placeholder UI (Shadcn) while waiting for Design. |

## 8. Success Metrics (KPIs)
- **Traffic:** ≥ 500 unique visitors in the first 3 months.
- **Conversion:** ≥ 1 high-quality lead (Job offer or Freelance contract).
- **Engagement:** Average session duration > 2 minutes.
- **Performance:** Lighthouse Score > 90 (Performance & SEO).
