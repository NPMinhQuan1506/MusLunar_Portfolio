# ? Definition of Done (DoD)

A story / feature is **DONE** only when all below criteria are satisfied:

| # | Area | Condition |
|---|-------|-----------|
| 1 | **Code** | Implemented, passes `lint` checks, and formatted (Prettier/Go Fmt). |
| 2 | **Review** | Code pushed to GitHub and checked by teammate (Dev checks logic, Designer checks UI). |
| 3 | **Responsive** | Verified on Mobile (iOS/Android) and Desktop sizes. |
| 4 | **Assets** | Images/3D Models are optimized (WebP, Compressed GLB) before usage. |
| 5 | **Documentation** | API/Schema docs updated in `docs/` folder (if changed). |
| 6 | **Deployable** | Merged to `main` branch. <br>- FE: Live on **Vercel**.<br>- BE: Running on **VPS**. |
| 7 | **Performance** | No console errors, fast loading speed. |
| 8 | **Analytics** | Tracking events are firing (e.g., "Role Selected"). |

**Optional (Nice-to-have for MVP):**
- [ ] Localization texts are fully updated (EN/VI).
- [ ] Accessibility (WCAG AA) compliance.
